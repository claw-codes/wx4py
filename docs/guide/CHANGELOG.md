# 更新日志

本文档记录项目各版本的重要变更。

版本号遵循语义化版本的基本原则：
- `PATCH`：修复问题、稳定性优化、兼容性调整
- `MINOR`：新增功能且保持兼容
- `MAJOR`：存在破坏性变更

## [0.2.1] - 2026-04-11

本版本重点完善群消息处理能力的发布形态，补充群消息转发说明、完整接口文档和更适合直接使用的示例体系。

### 新增

- 新增“监听指定群消息并转发给指定联系人或群”的正式发布说明，明确支持告警转发、值班通知和跨群消息同步场景。

- 新增一套面向小白的示例脚本写法：关键配置集中在顶部、中文步骤说明、统一 `main()` 入口。

### 优化

- 群消息处理统一收口为 `WeChatClient.process_groups()`，减少重复入口和维护成本。
- 群消息处理日志更加简洁，去掉了开发排障阶段的临时调试输出。
- 示例脚本更适合直接修改后运行，降低初次使用门槛。

### 文档

- 更新 README，补充“监听群消息并转发给指定联系人或群”的功能说明。
- 更新 wx4-skill 文档，修正过时调用方式，并补充群消息转发说明。
- 更新快速开始文档，补充接口文档入口。

## [0.2.0] - 2026-04-11

本版本新增群聊监听与 AI 自动回复能力，并增强微信隐藏到系统托盘后的恢复稳定性。

### 新增

- 新增多群聊消息监听能力，支持为多个群打开独立聊天窗口并持续监听新消息。
- 新增 `WeChatClient.process_groups()`，统一处理群聊监听、转发和自动回复场景。
- 新增 `MessageEvent` 和 `WeChatGroupListener`，封装群名、消息内容、群昵称、是否 @ 当前账号等监听事件信息。
- 新增发送队列，自动回复时串行发送消息，避免多个群同时回复导致窗口焦点抢占。
- 新增本库发送消息的回流忽略机制，降低机器人监听到自己回复后重复触发的风险。
- 新增群昵称读取能力 `group_manager.get_group_nickname()`，用于判断群消息是否 @ 当前账号。
- 新增通用 AI 模块 `AIClient`、`AIConfig`、`AIResponder`，支持 OpenAI 兼容接口、Responses API、Anthropic 风格接口和自定义回复回调。
- 新增 AI 自动回复示例和自定义 AI 接入示例，方便接入第三方模型或企业内部 AI 服务。

### 优化

- 多群监听采用单调度器分片轮询，减少多个窗口同时监听时的无效扫描。
- 群聊监听初始化流程优化为“搜索群聊、读取群昵称、打开独立窗口”逐个完成，减少重复搜索。
- AI 自动回复默认推荐 `reply_on_at=True`，普通消息只监听，只有被 @ 时才触发回复。
- 微信隐藏到托盘后的恢复逻辑优先使用原生托盘消息，不再依赖窗口强制显示，避免出现黑窗口。

### 修复

- 修复微信隐藏到系统托盘后无法正常唤出的问题。
- 修复部分环境下通过 Win32 强制显示窗口会出现黑屏但 UI 仍可拖动的问题。
- 修复系统任务栏隐藏或第三方 Dock 场景下依赖 UIA 托盘点击不稳定的问题。

### 文档

- 更新 README 功能列表，补充多群监听、@ 触发 AI 自动回复、群昵称读取和自定义 AI 接入说明。
- 更新 wx4-skill 文档，使 Claude Code / OpenClaw 能正确理解群聊监听与 AI 自动回复用法。
- 更新示例文档和官网版本信息。

## [0.1.4] - 2026-04-08

感谢 [yeafel666](https://github.com/yeafel666) 的贡献，本版本合并了他在微信窗口连接、UIA 辅助唤醒、搜索体验和窗口最小化相关的一系列改进。

### 优化

- 优化微信窗口连接和 UIAutomation 封装流程，改进登录界面辅助功能处理与主窗口等待逻辑。
- 优化微信 UIA 辅助唤醒与重启恢复流程，提升连接和恢复时的成功率。
- 优化搜索结果目标匹配优先级，并新增“最常使用”分组支持，使常用联系人和群聊更容易命中。
- 引入 `WeChatNotFoundError` 异常类，增强客户端连接失败时的错误语义。
- 调整发送任务结束后的窗口缩小时机：单次发送成功后缩小，批量发送在整批任务完成后再统一缩小。
- 添加微信窗口最小化能力，用于在自动化执行完成后更好地保护用户隐私。
- 完善 README 与 GitHub Pages 的演示视频展示方式，README 统一跳转官网查看，官网直接播放本地演示视频。
- 补充并整理项目展示内容，包括更新日志入口、致谢信息和开发者示例指引。

### 修复

- 清理发送链路中与窗口缩小相关的冗余传参，明确由编排层决定窗口行为。

## [0.1.3] - 2026-04-01

### 优化

- 重构消息发送流程，拆分重试、重连兜底和发送阶段执行逻辑，提升代码可读性和可维护性。
- 统一文本输入方式，普通消息和文件附带消息都改为通过剪贴板粘贴，降低 UI 自动化输入不稳定带来的问题。

- 将版本号维护收敛为单一来源 `src/_version.py`。

### 修复

- 修复目标群聊或会话不存在时仍然重复查找、重复重连的问题。
- 修复文件路径复制到剪贴板失败时仍可能继续发送的问题。
